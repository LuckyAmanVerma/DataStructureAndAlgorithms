Interview Crackers Graph Logic DSU / Union–Find

DSU ko kab use karein? (Hinglish Cheat Sheet)

Quick intuition + spotting tricks so you instantly know: “Is problem me DSU lagega ya nahi?”

⚡ 5-Second Spotting Checklist

  • Merge/Group — Problem bolta hai "yeh do cheezein same group me lao" ya "friends ko combine karo".
  • Connectivity — "A aur B connected hain?" ya "kitne components?".
  • Redundant Edge — "Kaunsa edge extra/cycle banata hai?".
  • Constraints as Edges — Equalities/Inequalities ko graph edges ki tarah treat karo.
  • Offline Queries — Batch me union karke answers nikaalne hain.

❌ Kab DSU nahi lagta

  • Shortest Path — Dijkstra/BFS/DFS better.
  • Topological Order — Kahn/DFS topo.
  • Directed Cycle Logic — DSU undirected cycles me best; directed ke liye graph algos use karo.
  • Dynamic online updates — Real-time add/remove edges? DSU sirf add/merge me strong; removals tough.

🎯 Classic DSU Use-Cases

  • Undirected graph me cycle detect karna / redundant connection find karna.
  • Connected components count / Number of Provinces.
  • Accounts Merge (emails ko group karna).
  • Similar String Groups / Friend Circles.
  • Minimum operations to connect network (extra wires = components-1 logic).
  • Grid Islands (2D me cells ka union) — optional vs DFS; large constraints me helpful.
  • Equations Possible? (a==b unions, a!=b conflicts check).
  • Kruskal’s MST (edges sort + union-find).

🧠 DSU Intuition (Hinglish)

  • Har node ka parent/leader hota hai. find(x) se leader milta.
  • union(a,b) matlab inke leaders same bana do ⇒ same group.
  • Path Compression = find ko turbo fast; Union by Rank/Size = tree shallow.
  • Answers mostly leaders par dependent hote: groups kitne, members kaunse, conflict ya cycle.

🔍 Spotting Keywords in Statements

  • “merge / connect / combine / link / same group / same set”
  • “how many groups/components”
  • “extra/duplicate edge”, “cycle formed”
  • “are A and B related/connected/equivalent?”
  • “constraints like a==b, a!=b”

🧩 Mini Decision Tree

  • Q: Kya relations/edges ko combine karke groups banana hai? → DSU
  • Q: Kya sirf shortest distance ya path chahiye? → Dijkstra/BFS
  • Q: Inequalities + equalities diye hain? → Pehle == union, phir != check.
  • Q: MST banana hai? → Kruskal + DSU
  • Q: Ek edge hattane se kya hoga? → DSU typical nahi (removal hard), alternate structure socho.

🧪 Common Interview Patterns + Example Prompts

  1. Redundant Connection: “Given undirected edges, find the first edge that forms a cycle.”
  2. Provinces/Connected Components: “Given adjacency matrix or edges, count components.”
  3. Make Network Connected: “Minimum operations to connect all computers.” Hint: extra edges ≥ components − 1.
  4. Accounts Merge: “Merge accounts with common email/phone.”
  5. Similar String Groups: “Strings similar → same group; count groups.”
  6. Equations Satisfiable: “Check if all a==b and a!=b can hold simultaneously.”
  7. Kruskal MST: “Minimum total cost to connect all nodes.”
  8. Grid Islands via DSU: “Number of islands / largest island after flip.”

🛠️ DSU Skeleton (C++)

struct DSU {
  vector p, sz;
  DSU(int n): p(n), sz(n,1){ iota(p.begin(), p.end(), 0); }
  int find(int x){ return p[x]==x ? x : p[x]=find(p[x]); } // path compression
  bool unite(int a, int b){
    a = find(a); b = find(b);
    if(a==b) return false;               // already same group
    if(sz[a] < sz[b]) swap(a,b);         // union by size
    p[b] = a; sz[a] += sz[b];
    return true;
  }
};

🚨 Pitfalls (Interview me bacho)

  • Indexing galat (0/1-based mix) ⇒ out of range.
  • Union by rank/size na lagaaya ⇒ TLE chances badh jaate.
  • Directed graph cycle ke liye DSU mat lagao — logic mismatch.
  • Edge removals required? DSU direct fit nahi; offline tricks ya DS data socho.

📝 Rapid Heuristics (Memorize karo)

  • Components Count = leaders ki distinct count.
  • Redundant Edge = unite() returns false ⇒ yahi answer.
  • Network Connect: answer = components − 1 (agar extraEdges ≥ components − 1).
  • Equations: pehle sab equalities union; baad me har inequality (a!=b) pe check find(a)==find(b) ⇒ conflict.
