Shortest Path Cheatsheet

Choosing between Dijkstra and Topological Sort

A quick reference for single-source shortest path. Includes when to use Dijkstra (non-negative weights, general graphs) versus Topological Sort (DAGs, works with negative weights if no cycles).

Undirected Unweighted → BFS (with parent) Directed/Undirected + non-negative weights → Dijkstra Any negative edge weights → Bellman–Ford All-pairs → Floyd–Warshall / Johnson
When to use

Dijkstra’s Algorithm

  • Graph type: Directed or undirected (general graphs).
  • Edge weights: Non‑negative only.
  • Cycles: Allowed (no negative cycles permitted).
  • Time: O((V + E) log V) with a min‑heap.

Typical uses: GPS/road routing, network latency minimization, flight/time/cost optimization where negatives don’t occur.

When to use

Topological Sort (DAG Shortest Path)

  • Graph type: DAG only (Directed Acyclic Graph).
  • Edge weights: Works with positive or negative (no cycles).
  • Cycles: Not allowed (acyclic by definition).
  • Time: O(V + E) (often faster than Dijkstra).

Typical uses: Project scheduling (CPM/PERT), compiler instruction scheduling, data pipelines, dependency resolution.

Quick guide

Rule of Thumb

  • If the graph is a DAG → prefer Topological Sort (linear time, handles negative weights).
  • If the graph has cycles but weights are non‑negative → use Dijkstra.
  • If any negative weights are present (unknown structure) → use Bellman–Ford.
  • Need all‑pairs shortest paths → try Floyd–Warshall or Johnson’s (if sparse).

Bonus: For undirected, unweighted shortest paths, use BFS with a parent array to reconstruct the path.

Note

Shortest Path (Undirected, Unweighted)

Use BFS with a parent array to reconstruct the path from the destination back to the source.

Complexity

At a glance

  • BFS: O(V + E)
  • Topological (DAG): O(V + E)
  • Dijkstra (heap): O((V + E) \log V)
  • Bellman–Ford: O(VE)
  • Floyd–Warshall: O(V^3)
Tip

Implementation hint

For Dijkstra, prefer a min‑heap/priority queue. For DAG shortest path, make sure you topologically sort first, then relax edges in that order.
