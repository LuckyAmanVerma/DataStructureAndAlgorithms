Sorting Decimal Numbers

When you need to sort decimal values (either fixed-precision numbers or real numbers in a known range), you can often beat O(n log n) comparison sorts by using digit- or bucket-based methods.

1. Radix Sort (for fixed-precision decimals)

  • How it works
    • Processes numbers digit by digit (least significant to most significant)
    • At each pass, distributes elements into 10 “buckets” by their current digit and then collects them back
  • Time complexity
    • O(d · (n + b)) where d = number of decimal digits, b = base (10)
    • For fixed d, this is effectively O(n)
  • When to use
    • Keys have a known, limited number of decimal places
    • You can afford O(n + b) extra space for buckets
    • You need a stable, linear-time sort without comparisons

2. Bucket Sort (for real decimals in [0, 1) or any known range)

  • How it works
    • Divides the range into k equal-width buckets
    • Distributes each element into its bucket based on its value
    • Sorts each bucket (often with insertion sort) and concatenates
  • Time complexity
    • Best/Average: O(n + k) → with k ≈ n and uniform data, average is O(n)
    • Worst: O(n²) if all values land in one bucket
  • When to use
    • Values are uniformly distributed over a known interval (e.g., [0, 1))
    • You want expected linear time on floats or real decimals
    • You can choose k proportional to n

Why Not Comparisons?

  • Comparison sorts (Quick, Merge, Heap) are bound by Ω(n log n)
  • Digit- and bucket-based methods exploit the numeric structure to reach O(n) under the right conditions
