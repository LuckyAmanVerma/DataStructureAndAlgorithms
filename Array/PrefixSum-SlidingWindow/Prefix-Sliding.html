🔍 Minimum Size Subarray Sum ≥ k

✅ When to Use Only Sliding Window

  • Array has only positive integers
  • Sum is monotonic: adding more elements always increases sum
  • You want the smallest length with sum ≥ k
Sliding Window = Perfect Fit because:
  • Grow window while sum < k
  • As soon as sum ≥ k, shrink from left to minimize length
  • Everything in O(n) time

⚠️ When to Use Prefix Sum + Binary Search (Combo Approach)

  • When interviewer specifically asks:
    “Use prefix sum”
    “You may need to perform multiple sum range queries”
  • When array contains negative numbers (sliding window fails as sum can go up/down)
  • When your goal is:
    Find the smallest subarray with sum ≥ k
    AND optimize for non-monotonic values
Prefix Array + Binary Search gives O(n log n) solution for these cases.

Prefix Sum vs Sliding Window: Pehchaan Ke Tips & Tricks (Hinglish)

1. Jab Prefix Sum Use Karna Hai

  • Multiple range‐sum queries: Agar repeatedly “sum of nums[l…r]” puchna ho, prefix sum natural fit hai.
  • Sum equals K ya divisible by K: “Count of subarrays with sum = k” ya sum%k==0 me aapko prefix+hashmap use karna hota hai.
  • Negative elements allowed: Jab array me negative values ho, sliding window break ho sakta hai, tab prefix sum + map/BS approach safe rehta hai.
  • Offline queries: Agar pehle se saare queries mile hue ho (jaise l,r pairs), precompute prefix aur phir O(1) me answer do.

2. Jab Sliding Window Use Karna Hai

  • Fixed window size (k): “Max sum/average of size k” instantly sliding window bana kar solve hota hai.
  • Positive elements only: Sum monotonically badhega; shrink/expand logic easily work karega.
  • Target sum ≥ K with positive numbers: “Minimum length subarray with sum ≥ k” sliding window O(n) me karega, prefix sum O(n²) se better.
  • Longest/shortest substring patterns: Strings me “longest unique‐char substring”, “minimum window containing pattern” sliding window+two‐pointers se best solve hote hain.

3. Recognition Heuristics

  • Question me ‘contiguous subarray’ aur “sum” dono nazar aaye → prefix sum à la range queries, ya sliding window à la grow‐shrink.
  • Problem bol raha ho “count” → prefix sum+map for counts, kyunki sliding window se count tricky hota hai.
  • Problem bol raha ho “max/min length” aur only positives → sliding window.
  • Constraints me negative numbers mention ho → sliding window unreliable; prefix sum + hashmap/BST approach dekho.
  • Multiple sum queries vs single‐pass optimization: Jitna zyada single traversal me solution ho, sliding window hi dhundho.

4. Quick Decision Flow

  1. “Fixed size k?” → Sliding window
  2. “Sum exactly k or count subarrays?” → Prefix sum + HashMap
  3. “Sum ≥ k, minimize length, all positive?” → Sliding window
  4. “Negative elements or complex ≤/≥ constraints?” → Prefix sum + Binary Search/Map
